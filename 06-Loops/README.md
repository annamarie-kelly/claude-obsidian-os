# `06-Loops/` — Open items dashboard

This folder is the **loops ledger** for your vault: a derived, queryable view of every unfinished `- [ ]` item scattered across your notes, plus session state for the companion Loops UI (`tools/loops-ui/`).

## Files

| File | Written by | Purpose |
|---|---|---|
| `loops.json` | scanner + UI | Canonical ledger of every open loop. Re-generated by `npm run refresh-loops`; mutated in place by the UI as you triage / schedule / close things. |
| `open-loops.md` | `/loops` slash command | Human-readable snapshot for reading in Obsidian. Priority-sorted, grouped by domain. Never edit this file by hand. |
| `events.log.jsonl` | UI + scanner | Append-only audit log of every mutation. Each line is one event. Used for debugging divergence between UI and vault. |
| `boundary_log.json` | UI (mirror) | Capacity-gate overrides, scope acknowledgements, and other self-protection events the UI writes so external tooling can read them. |
| `tend-export.json` | UI (mirror) | Checkpoints and weekly patterns, mirrored from localStorage. |
| `stakeholder-window.json` | UI (mirror) | Weekly completed / started / blocked / flags window for the primary stakeholder. |
| `calendar-today.json` | **you** | Calendar events for the week. Populate however you like — a cron pulling from Google Calendar, a MCP tool writing from a chat session, or hand-edited JSON. See schema below. |

## `calendar-today.json` schema

```json
{
  "lastSynced": "2026-04-22T08:00:00Z",
  "events": [
    {
      "id": "unique-event-id",
      "date": "2026-04-22",
      "title": "Team standup",
      "startMinute": 540,
      "endMinute": 570
    }
  ]
}
```

- `startMinute` / `endMinute` are minutes-from-midnight (e.g. `540` = 9:00 am).
- `date` is ISO "YYYY-MM-DD" in local time.
- The Loops UI reads this file read-only and places events on the week canvas.

## Generating loops

From the loops-ui directory:

```bash
npm run refresh-loops       # scan the vault, update loops.json
npm run refresh-loops:dry   # preview without writing
```

The scanner walks `00-Inbox`, `01-Building`, `02-Thinking`, `03-Working`, `04-Living`, `05-Relating` and picks up every `- [ ]` that isn't checked off. Priority is inferred conservatively (everything defaults to P3 — you promote things manually via the triage gate).
