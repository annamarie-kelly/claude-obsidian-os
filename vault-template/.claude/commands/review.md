# Weekly Review

The weekly metacognition pass. Run triage and loops first — this is about synthesis, not processing.

$ARGUMENTS

## Steps

1. **Scan recent activity** — check all notes modified in the past 7 days across the vault. Summarize what was worked on by domain (Building, Thinking, Working, Living, Relating).

2. **Task completion review** — how many `- [ ]` items were closed this week vs. created? Are tasks accumulating faster than they're being resolved? Flag any concerning trends.

3. **Surface emerging patterns** — from recent notes, identify insights worth distilling:
   - New patterns to extract (suggest claim-style titles)
   - Seed patterns ready to promote to `growing`
   - Growing patterns ready to promote to `evergreen`
   - Suggest updates to relevant `_patterns.md` files

4. **Project health** — for each active project in `01-Building/`, check if recent notes mention it. Flag projects with no activity this week.

5. **Life admin check** — scan `04-Living/` for overdue or neglected items. Health appointments, financial tasks, and recurring obligations that might be slipping.

6. **Orphan check** — find notes with zero inbound links that might need connecting. Suggest specific `[[wikilinks]]` to add.

7. **Inbox check** — if there are unprocessed items in `00-Inbox/`, flag them (but don't process — that's `/triage`).

8. **Create review note** — save a summary to `07-Archive/` using the weekly-review template with:
   - What domains got attention this week
   - Patterns extracted or promoted
   - Tasks closed / opened / stale
   - Connections made
   - What felt neglected
   - Energy/wellbeing observations

Present findings as a summary and ask before making changes.
